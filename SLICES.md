# Slices

Each slice is a directory under [.claude/rules/](.claude/rules/). Pick one or more to symlink into your consumer repo's `.claude/rules/`.

- **autonomy** — agent operating posture: when to act vs. confirm, bash discipline, parallel subagents, permission-prompt hygiene. Most consumers want this.
- **git** — git workflow conventions: don't stage, hand off commit messages, submodule branch hygiene.
- **meta** — for repos that author their own Claude rules, skills, or commands: where new rules go, when to propose them, markdown style, dependency direction.
