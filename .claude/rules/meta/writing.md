# Writing

Markdown in this repo — rules, skills, commands, READMEs — loads into every consumer's session. Bloat propagates and recurs as token cost forever.

State each point **once**. Don't restate the same idea in prose, then bullets, then a worked example, then a closing sentence. Before saving any markdown file here, reread it and delete any paragraph or list whose removal wouldn't change what the reader knows or does.

A worked example earns its keep only when the rule has a non-obvious edge case or input shape (YAML, command syntax) that prose can't convey. Two main forms placed side-by-side under a header is not that.
